# スクリーニングシート 読み方ガイド

---

## シートの構造

| 列 | 項目名 | 誰が書くか |
|----|--------|-----------|
| A | 銘柄コード | ユーザーが入力 |
| B | 会社名 | ユーザーが入力 |
| C | 業種 | ユーザーが入力 |
| D〜AA | 各種判定・スコア | スクリプトが毎日自動更新 |

A〜C列はスクリプトが絶対に触らない。D列以降は毎日上書きされる。

---

## まず「総合判定（I列）」だけ見ればよい

スクリーニングの最終結果。**最初はここだけ確認すれば十分。**

| 判定 | 意味 | 総合スコアの目安 |
|------|------|----------------|
| **AA** | 最上位 | 95以上 |
| **A** | 上位 | 90〜94 |
| **B** | 良好 | 80〜89 |
| **C** | 合格 | 70〜79 |
| **D** | 条件付き通過。安全条件は満たすが収益力が相対的に弱い | 50〜69 |
| **FAIL** | 不合格。安全系のゲートで落ちた | — |
| **ERROR** | データ取得失敗。判定不能 | — |

**投資の優先度は B以上を中心に、C・D は補助的に確認する流れが基本。**

---

## 判定のしくみ：2段階構造

スクリーニングは「**ゲート**」と「**スコア**」の2段構えになっている。

```
[ゲート1] 投機性判定 ──┐
[ゲート2] DD判定     ──┼── 1つでもFAIL → 総合判定 = FAIL（問答無用）
[ゲート3] デュレーション ─┤
[ゲート4] Q5判定     ──┘
         ↓ 4つ全部PASS
[スコア計算]
 デュレーションスコア × 60% + Q5スコア × 40% = 総合スコア
         ↓
 95以上→AA / 90以上→A / 80以上→B / 70以上→C / 50以上→D / 50未満→FAIL
```

重要なのは、**投機性とDDの2つは「落とすためのゲート」**（FAILなら即除外）で、
**デュレーションとQ5は「優劣をつけるためのスコア源」**という役割分担になっている点。

---

## 4つのゲート（E〜H列）の意味

### E列：投機性判定

「この銘柄はギャンブル的な動きをしていないか」を評価する。
株価のブレ幅（固有ボラ）・市場連動性（ベータ）・財務ストレス（負債÷資産、CFの薄さ）を合成したスコア。

- **PASS** = 値動きが比較的安定していて財務も健全
- **FAIL** = ボラが高すぎる、またはストレス指標が大きい

### F列：DD判定（デフォルト・ディスタンス）

「この会社は倒産・債務不履行までどれだけ距離があるか」を定量的に評価する。
時価総額・負債・ボラを使ったMertonモデルで計算。数値が大きいほど安全。

- **PASS** = 倒産リスクが十分に低い
- **FAIL** = 財務的に危険水域に近い
- 無借金企業（理由コード `NO_DEBT`）は自動的にPASS、DDスコア=100になる

### G列：デュレーション判定

「この会社はキャッシュを稼ぎ続けられるか」を評価する。
FCF利回り・営業CF利回り・設備投資負担・CF安定性を総合評価。

- **PASS** = キャッシュ創出力が全銘柄の上位水準
- **FAIL** = FCFや営業CFが相対的に低い、または設備投資の負担が重すぎる

### H列：Q5判定（成長品質）

「この会社は質の高い成長をしているか」を評価する。
総資産成長率・ROE・ROE変化率・資産効率を総合評価。

- **PASS** = 成長の質が相対的に上位水準
- **FAIL** = ROEが低い（5%未満）、ROEが前年より悪化、資産だけ膨らんでいるなど

## スコア列（J〜N列）の読み方

スコアはすべて **0〜100 のパーセンタイル**。
「今回スクリーニングした全銘柄の中で上位何％か」を示す相対評価。

| 列 | スコア | 高い = |
|----|--------|--------|
| J | 投機性スコア | 安定・低リスク（低ボラ、低財務ストレス） |
| K | DDスコア | デフォルトリスクが低い |
| L | デュレーションスコア | キャッシュ創出力が強い |
| M | Q5スコア | 成長の質が高い |
| N | 総合スコア | デュレーション60% + Q5スコア40% の加重平均 |

スコアは相対評価なので、同じ会社でも母集団が変わればスコアも変わる。
絶対的な良し悪しではなく、**今回の対象銘柄の中での順位**として読む。

---

## 数値指標（O〜V列）の読み方

| 列 | 項目 | 見方 |
|----|------|------|
| O | ベータ(252日) | 市場全体に対する感応度。1.0が市場と同じ。大きいと市場に追随して動く |
| P | 固有ボラ(252日) | 市場を除いた株価固有のブレ幅（年率）。小さいほど安定 |
| Q | DD生値 | デフォルト距離の実数。目安：2以上は安全圏、1を割ると注意水域 |
| R | FCF利回り | FCF ÷ 時価総額。高いほど株価に対してキャッシュを稼いでいる |
| S | 営業CF利回り | 営業CF ÷ 時価総額。FCFより安定した本業のキャッシュ力 |
| T | 設備投資負担 | 設備投資 ÷ 営業CF。低いほどCFが設備に食われていない（0.5以下が目安） |
| U | 総資産成長率(1年) | 総資産の前年比増加率。高すぎると水膨れの可能性もある |
| V | ROE(TTM) | 直近12ヶ月の自己資本利益率。高いほど株主資本を効率よく稼いでいる |

## 補助列（W〜AA列）

| 列 | 項目 | 内容 |
|----|------|------|
| W | データ充足率 | 0〜100。100が全データ揃い。低いほど判定精度が落ちる |
| X | 理由コード | 判定理由のコード（複数ある場合はカンマ区切り）。下表参照 |
| Y | 理由テキスト | 理由コードを日本語で要約 |
| Z | エラーメッセージ | データ取得失敗時のエラー種別 |
| AA | 更新日時(JST) | このデータが書き込まれた日時 |

### 主な理由コード

| コード | 意味 |
|--------|------|
| `NO_DEBT` | 無借金企業。DD は自動PASS、DDスコア=100 |
| `Q5_NEG_DROE` | ROEが前年より悪化 |
| `Q5_LOW_ROE` | ROEが5%未満（絶対水準が低い） |
| `Q5_HIGH_ASSET_GROWTH` | 総資産が急拡大（15%超） |
| `DUR_LOW_FCF` | FCF利回りがマイナス |
| `DUR_HIGH_CAPEX` | 設備投資が営業CFを超えている |
| `SPEC_HIGH_IVOL` | 固有ボラが高い |
| `DATA_PRICE_SHORT` | 価格データの日数が不足 |
| `PARTIAL_MODEL_DATA` | 財務データの一部が欠損 |
| `PARSE_ERROR` | 銘柄コードが不正 |

## 知っておくと便利なパターン

**「ゲートは4つ全部PASSだが総合がD」**
デュレーションスコアとQ5スコアが両方とも50〜69の範囲に収まっているケース。
安全条件はクリアしているが、同業他社と比べてキャッシュ力・成長力のどちらかが弱い。

**「デュレーション/Q5の片方がFAILだが総合がCやD」**
FAILしたゲートのスコアでも、もう一方が高ければ加重平均で総合スコアが50以上になることがある。
「どのゲートが落ちているか」を理由コードで必ず確認する。

**「理由テキストが空白」**
FAILの理由に該当するコードが理由コード欄に登録されていない場合に起きる。
データ的には問題なく処理されている。

## 運用スケジュール

| タイミング | 更新内容 |
|-----------|---------|
| 毎日（月〜日）JST 17:15頃 | 価格データ・ベータ・固有ボラ・総合判定 |
| 毎週日曜日（同時刻） | 上記に加えて財務データ（CF・負債・資産・ROEなど）も更新 |

財務データは四半期ごとにしか変わらないため週1更新で十分。
月〜土は株価ベースのリスク指標だけを最新化する。

## よくある疑問

**Q. FAILでも良い銘柄はあるか？**
ゲート設計上、FAILは「現時点でリスク指標が高い」という意味。理由コードを見て原因を確認する。財務が改善すれば翌週以降にPASSに転じる可能性はある。

**Q. DとCではどちらを優先するか？**
C以上を優先。Dは安全条件を満たしているが収益力・成長力が相対的に弱い。同じ業種で比較するとわかりやすい。

**Q. スコアが同じならどちらが良いか？**
FCF利回り・設備投資負担・ROEなどの個別指標を比較する。特に設備投資負担が低い銘柄は「稼いだCFを自由に使える」状態を示している。

**Q. NAはどういう状態か？**
データが不足していてそのゲートを判定できなかった状態。危険とは限らないが、データ充足率（W列）も合わせて確認する。
